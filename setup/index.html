ABC DAO Developer Onboarding

Fedora + Claude Code + OpenClaw Setup Guide

← Alpha / Docs / Setup

Welcome to ABC DAO. This guide walks you through setting up a development environment on your MacBook using Fedora Linux, Claude Code, and OpenClaw.

Step 0 — Determine Your MacBook Type

Before we start, you need to know if your MacBook has an Intel or Apple Silicon (M1/M2/M3) chip.

On macOS:

  1. Click the Apple menu → "About This Mac"
  2. Look for "Chip" or "Processor"
    • If it says Apple M1, M2, M3, etc. → You have Apple Silicon
    • If it says Intel Core i5, i7, i9, etc. → You have Intel

The installation process differs significantly between these two architectures.

Step 1 — Install Fedora Linux

Option A: Apple Silicon MacBook (M1/M2/M3)

You'll use Fedora Asahi Remix, a Fedora distribution specifically built for Apple Silicon Macs.

Official Resources:

  • Homepage: fedora-asahi-remix.org
  • Installation Guide: Fedora Asahi Remix Docs

Installation Steps:

  1. Backup your Mac — This will resize your macOS partition or replace it entirely. Back up anything important.
  2. Open Terminal on macOS and run: 
    curl https://alx.sh | sh
  3. Follow the interactive installer — It will:
    • Ask how much disk space to allocate to Fedora
    • Download and install the bootloader
    • Set up Fedora Asahi Remix
  4. Reboot — Hold the power button during startup to access the boot picker, then select Fedora.
  5. Complete first-boot setup — Create your user account, set timezone, etc.
Notes for Apple Silicon:
  • WiFi, Bluetooth, and speakers work out of the box on most models
  • Battery life is excellent
  • Some features (Touch Bar, certain webcam models) may have limited support
  • Check the Device Support page for your specific model

Option B: Intel MacBook

You'll use standard Fedora Workstation.

Official Resources:

  • Download: fedoraproject.org/workstation/download
  • Installation Guide: Fedora Install Guide

What You'll Need:

  • USB drive (8GB minimum)
  • Fedora Workstation ISO (download the x86_64 version)
  • Fedora Media Writer or balenaEtcher to create the bootable USB

Installation Steps:

  1. Download Fedora Workstation ISO — Go to fedoraproject.org/workstation/download and download the x86_64 ISO.
  2. Create a bootable USB — Use Fedora Media Writer or balenaEtcher. Insert your USB drive and write the ISO to it.
  3. Boot from USB — Restart your Mac and hold the Option (⌥) key immediately. Select the USB drive (usually labeled "EFI Boot").
  4. Install Fedora:
    • Select "Install to Hard Drive"
    • Choose your language and keyboard
    • Click "Installation Destination" and select your target disk
    • For dual-boot: Click "Custom" and create a new partition; leave macOS partitions alone
    • For full install: Select "Automatic" (this will erase macOS)
    • Begin installation and create your user account
  5. Post-install for Intel Macs — WiFi should work, but you may need additional drivers. Run: 
    sudo dnf update

Step 2 — Initial Fedora Setup

Once you're booted into Fedora, open a terminal and run these commands:

# Update your system
sudo dnf update -y

# Install essential development tools
sudo dnf install -y git curl wget nodejs npm

# Verify Node.js version (should be 18+ for Claude Code)
node --version

# If Node.js is too old, install a newer version via nvm:
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
nvm install 22
nvm use 22

Step 3 — Install Claude Code

Claude Code is Anthropic's official CLI for Claude. It's the foundation for your AI-assisted development workflow.

Official Resources:

  • Documentation: docs.anthropic.com/en/docs/claude-code
  • npm package: @anthropic-ai/claude-code

Installation:

# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Verify installation
claude --version

First-Time Setup:

# Start Claude Code
claude

# On first run, it will prompt you to authenticate
# You'll need an Anthropic API key from https://console.anthropic.com/
# Or sign in with your Anthropic account

When Claude Code starts, it will ask you to authenticate. You can either:

  1. Enter an API key directly
  2. Sign in via browser (recommended for personal use)

Step 4 — Use Claude Code to Install OpenClaw

Now for the fun part — let Claude Code do the heavy lifting.

Start Claude Code:

claude

Then paste this prompt:

I need to install OpenClaw, the AI agent gateway. Please:

1. Check if I have the prerequisites (Node.js 18+, npm)
2. Install OpenClaw globally via npm
3. Run `openclaw init` to set up the initial configuration
4. Help me configure it for basic use with Claude as the AI backend

Walk me through each step and explain what's happening.

Claude Code will:

  • Verify your Node.js version
  • Run npm install -g openclaw
  • Guide you through openclaw init
  • Help configure your API keys and preferences
What to Expect During Setup:

OpenClaw init will ask you about:

  • AI Provider: Select Anthropic/Claude
  • API Key: Enter your Anthropic API key
  • Workspace: Where to store agent files (default: ~/.openclaw/workspace)
  • Channels: Which messaging surfaces to enable (Discord, Telegram, etc.)

Step 5 — Verify Everything Works

After installation, test your setup:

# Check OpenClaw is installed
openclaw --version

# Start OpenClaw gateway
openclaw gateway start

# In another terminal, check status
openclaw status

Start a Claude Code session in your workspace:

cd ~/.openclaw/workspace
claude

You should now have a working development environment with:

  • Fedora Linux running natively on your MacBook
  • Claude Code for AI-assisted development
  • OpenClaw for agent orchestration

Troubleshooting

"Command not found" after installing npm packages

Your PATH may not include npm global binaries. Add this to your ~/.bashrc:

export PATH="$HOME/.npm-global/bin:$PATH"

Then run source ~/.bashrc.

WiFi not working (Intel MacBook)

Some Intel MacBooks need the Broadcom WiFi driver:

sudo dnf install broadcom-wl

Poor battery life (Intel MacBook)

Install TLP for power management:

sudo dnf install tlp tlp-rdw
sudo systemctl enable tlp

Apple Silicon: Missing features

Check the Asahi Linux feature support matrix: Feature Support
